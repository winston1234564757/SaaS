# BOOKIT — Killer Features 2026–2027
### *Від інструменту запису до платформи, без якої майстер не може жити*

> Документ для стратегічного обговорення. Тут немає "зробити красивіше" — тут тільки фічі, які змінюють економіку майстра або поведінку клієнта фундаментально.

## 🧠 1. BOOKIT AI ENGINE — Мозок платформи

**Ідея:** Не чатбот. Не GPT-обгортка. Чиста математика на реальних даних Bookit.

### 1.1 Churn Prediction — «Клієнт іде»
```
Алгоритм: exponential decay на інтервалах між візитами
Вхід: середній інтервал клієнта (наприклад, кожні 23 дні)
Якщо пройшло > 1.8× середнього інтервалу → тривога
```
**Що бачить майстер:** У CRM з'являється тег `⚠️ Ризик відтоку` і кнопка «Нагадати».
**Що отримує клієнт:** Персональне повідомлення в Telegram/SMS від майстра:
*«Аня, минуло 35 днів з твого останнього манікюру 💅 Є вільний четвер 15:00 — записатися?»*
**Вплив:** +15–25% повернення "сплячих" клієнтів за дослідженнями аналогів.

---

### 1.2 Smart Rebooking — Автоматичний наступний запис
```
Після кожного completed запису → predict_next_visit(service_type, client_history)
Манікюр → через 21 день
Фарбування → через 45 днів
Брови → через 28 днів
```
**Що відбувається:**
- За 3 дні до розрахункової дати клієнту приходить пуш: *«Час для манікюру! Твій улюблений майстер Ірина вільна у вівторок і п'ятницю»*
- Клієнт записується в 2 тапи прямо з нотифікації (deeplink)
- Майстер нічого не робить — система сама заповнює розклад

**Монетизація для Bookit:** функція Pro, +60% утримання клієнтів = майстер не піде з платформи.

### 1.3 Revenue Pulse — Щоденний брифінг
Щоранку о 9:00 майстру приходить Telegram-повідомлення:
```

Сьогодні: 4 записи · 2 800 ₴ очікується
Цей тиждень: на 12% краще ніж минулий
⚠️ Четвер порожній — 3 клієнти "on hold" можуть записатися

```
**Це не email-дайджест. Це живий пульс бізнесу прямо в месенджері.**

## ⚡ 2. FLASH DEALS — Миттєве заповнення розкладу

**Проблема:** Майстер має порожній слот завтра о 14:00. Зараз він просто втрачає гроші.

**Рішення:** Кнопка `⚡ Заповнити слот` в розкладі.

```
Майстер: обирає слот → встановлює знижку (10–50%) → натискає
Система: за 3 секунди відправляє push всім клієнтам цього майстра
         які бували в схожий час + не мають запису на завтра

Повідомлення: «⚡ Флеш-пропозиція від Ірини!
               Манікюр завтра 14:00 — 280 ₴ замість 400 ₴ (до 20:00 сьогодні)
               [Записатися зараз]»
```

**Обмеження:** 2 флеш-пропозиції на місяць на Starter, необмежено на Pro.
**Монетизація Bookit:** комісія 3% з флеш-записів (клієнт вже платить онлайн).
**Ефект:** майстри заповнюють 20–35% раніше порожніх слотів.

## 🎭 3. BEAUTY PASSPORT — Портативна ідентичність клієнта

**Ідея:** Кожен клієнт Bookit має єдиний профіль, який подорожує між майстрами.

```yaml
Паспорт Марії Коваленко:
  preferred_gel: "OPI Bubble Bath"
  nail_shape: "мигдаль, довгі"
  allergies: ["aceton", "метакрилат"]
  skin_type: "суха шкіра рук"
  color_history: ["#E8C5C8", "#FF6B6B", "french"]
  scalp_sensitivity: null
  visits_total: 47
  avg_spend: 890 ₴
  bookit_score: 94  # reliability score
```

**Як це працює:**
1. Клієнт заповнює паспорт один раз у `/my/profile`
2. При записі до нового майстра — вибирає "поділитися паспортом"
3. Майстер бачить повну картину ДО прийому
4. Після візиту майстер може доповнити: *«Нанесла PolyGel nude»*

**Чому це game-changer:**
- Клієнт більше не пояснює "у мене алергія на..." 47 разів
- Майстер готується до прийому з реальними даними
- **Lock-in ефект:** клієнт з заповненим паспортом ніколи не піде з Bookit

## 💰 4. DEPOSIT SYSTEM — Захист від no-show

**Факт:** 15–25% записів у сфері краси — no-show або пізнє скасування. Це катастрофа для майстра.

**Рішення:** Майстер включає депозит у налаштуваннях:
```
Депозит: 20% / 50% / фіксована сума
Повернення при скасуванні: за 24+ год — повна, за 12-24 год — 50%, менше 12 год — 0
```

**UX для клієнта:**
- При записі: *«Для підтвердження потрібен депозит 80 ₴ (20% від вартості). Решта — у майстра.»*
- Кнопка «Оплатити депозит» → WayForPay / Mono
- При приході: майстер відмічає — решта готівкою або онлайн

**Монетизація Bookit:** 1.5% з кожного депозиту (набагато менше ніж stranded дохід майстра від no-show).

**Очікуваний результат:** зниження no-show з 20% до 3–5%. Це +15–20% виручки майстра без жодного нового клієнта.

## 👥 5. GROUP & EVENT BOOKINGS — Весілля, дівич-вечір, випускний

**Проблема:** Наречена хоче записати себе + 4 подруг до одного майстра на один ранок. Зараз це неможливо через Bookit.

**Рішення:** Тип запису «Подія»

```
Організатор: вибирає послуги + кількість місць + дату
Отримує: посилання-запрошення
Учасниці: відкривають, бачать "Весілля Оксани — 22 березня 9:00"
           вибирають свою послугу, платять (або організатор платить за всіх)
Майстер: бачить блок-бронювання з розписаним графіком по клієнтах
```

**Бонус:** Майстер може мати ставки для групових замовлень (-5% від 3 осіб).

**Ринок в Україні:** лише весіль — ~250,000 на рік. Кожне весілля = 3–8 записів.

## 📸 6. SMART PORTFOLIO AI — Портфоліо, яке продає

**Ідея:** Майстер завантажує фото роботи → AI робить решту.

```
Що робить AI:
1. Auto-crop & enhance (без фільтрів, просто light correction)
2. Auto-tagging: "french манікюр", "гель-лак", "нарощування", "рожевий"
3. SEO alt-text генерація для Google Image Search
4. Auto-watermark: ім'я + bookit.com.ua/slug в кутку
5. Визначає "best performer" — які фото → більше кліків → більше записів
```

**Для клієнта:** фільтр у публічному профілі майстра по тегу ("покажи тільки french") — як Pinterest.

**Технологія:** Cloudinary AI transformations / AWS Rekognition. Без власної ML-моделі.

## 🏆 7. BOOKIT SCORE — Репутація клієнта

**Ідея:** Як кредитний скоринг, але для краси.

```
Bookit Score: 1–100

Фактори:
+ Своєчасні скасування (>24 год)          +5
+ Завершені записи без no-show            +3

**Що це дає майстру:** може встановити мінімальний Score для запису.
**Що це дає клієнту:** з Score 90+ — пріоритетні сповіщення про відкриті слоти, доступ до flash deals першим.
**Гейміфікація:** "Твій Score 87 — ти в топ-15% клієнтів Bookit 🥇"

**Чому це спрацює:** Є тисячі майстрів, які хочуть навчитися у конкретних людей, чиї роботи вони бачать в Instagram. Bookit — місце де ці люди вже є.

## 🌍 9. DYNAMIC PRICING ENGINE

**Ідея:** Як авіаквитки. Ціна залежить від попиту.

```
Режими (майстер обирає):
1. Статична ціна (зараз)
2. Пікові години +15% (п'ятниця 16:00–20:00, субота)
3. Тихі години −10% (середа 10:00–13:00)
4. Last-minute discount (слот через 4 години — автоматично -20%)
5. Early-bird discount (запис за 2+ тижні — -7%)
```

**Відображення:** клієнт бачить "*Ціна у п'ятницю: 480 ₴. У середу 10:00: 360 ₴ ✨*"
**Ефект:** вирівнювання завантаженості тижня, ріст виручки в піки, ліквідація простоїв.

## 💬 10. AI RECEPTIONIST — Автоматична відповідь клієнтам

**Проблема:** Клієнт пише у Instagram о 23:00 «Є час у суботу?» — майстер спить, клієнт іде до конкурента.

**Рішення:** Bookit Telegram/Instagram bot що відповідає замість майстра:

```
Клієнт в Instagram: «Є час на манікюр у суботу?»

AI Receptionist (миттєво):
«Привіт! 👋 Ірина наразі недоступна, але я можу допомогти.
У суботу є вільний час: 10:00, 13:00, 16:00.
Записатися онлайн: bookit.com.ua/iryna
Там можна обрати послугу і час прямо зараз 😊»
```

**Технологія:** Meta Messaging API + Telegram Bot + наш розклад.
**Монетизація:** Pro фіча. Майстер підключає свій Instagram/Telegram.

## 📊 11. COMPETITIVE INTELLIGENCE — Анонімна аналітика ринку

**Ідея:** Агрегуємо дані всіх майстрів (анонімно) → даємо кожному бенчмарк.

```
Твоя аналітика vs. ринок (Київ, Манікюр):

Середній чек:         890 ₴   vs. 720 ₴ по ринку    ✅ +24%
Конверсія перегляд→запис: 8%  vs. 12% по ринку      ⚠️
Повторні клієнти:    68%      vs. 54% по ринку       ✅
Заповненість пн-ср:  45%      vs. 71% по ринку       ❌ Є проблема

💡 Порада: майстри з вищою заповненістю пн-ср в середньому мають 1.8 послуги в описі
            і публікують портфоліо щонайменше 2 рази на тиждень.
```

**Це не шпіонаж — це дзеркало.** Майстер не знає хто конкуренти, але знає де він у ринку.

## 🎁 12. BOOKIT SUBSCRIPTIONS — Підписка для клієнта

**Ідея:** Майстер продає клієнту місячну підписку на послуги.

```
Ірина пропонує: «Клаб постійного клієнта»
490 ₴/місяць = 2 манікюри (vs. 800 ₴ окремо)

Клієнт платить через Bookit → Ірина отримує передплату
Клієнт отримує: 2 бронювання на місяць без черги + -15% на товари
```

**Чому це революція для майстра:** передбачуваний MRR (monthly recurring revenue) замість хаотичного доходу.
**Чому клієнт платить:** економія + гарантія місця у завантаженого майстра.
**Bookit бере:** 5% комісії + транзакційну комісію платіжної системи.

## 🔄 14. SMART WAITLIST — Черга на запис

**Проблема:** Клієнт хоче до популярного майстра, але всі слоти зайняті.

**Рішення:**
```
Клієнт: натискає "Стати в чергу" на зайнятий слот
        вибирає пріоритет: "тільки ця дата" / "будь-яка дата цього тижня"

Автоматика:
- Скасування запису → система перебирає waitlist
- Першому в черзі → push + SMS «Відкрився слот! 15 хвилин щоб підтвердити»
- Не підтвердив → наступний в черзі
```

**Вплив:** заповнення 80-90% скасованих слотів замість поточних 20-30%.

## 📱 16. MINI-APP в Telegram

**Ідея:** Повноцінний BookingFlow всередині Telegram без виходу з месенджера.

```

## 💡 НАЙБІЛЬША СТАВКА (особисто від мене)

**Telegram Mini App + Flash Deals + Deposit** — це не три фічі. Це одна ідея.

> *Майстер прокидається, бачить що завтра порожній слот, натискає ⚡, — і за 30 хвилин слот заповнений клієнтом, який вже оплатив депозит прямо в Telegram. Майстер не зробив нічого, крім одного тапу.*

Це той момент, коли продукт стає магічним. Коли майстер показує телефон подрузі і каже: "дивись як це працює". Це word-of-mouth, якого не купиш за рекламу.
