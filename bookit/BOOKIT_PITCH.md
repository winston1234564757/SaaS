## Що таке Bookit?

**Bookit** — українська SaaS-платформа для онлайн-запису клієнтів у б'юті-індустрії.

Кожен майстер отримує персональну сторінку (`bookit.com.ua/свій-slug`) — повноцінний інструмент бізнесу, не просто форму запису. Клієнт відкриває посилання, бачить послуги, вибирає час, записується й отримує підтвердження на email. Майстер миттєво бачить запис у Telegram.

---

## Проблема

Б'юті-майстер в Україні сьогодні:

- Приймає записи в **Instagram Direct** і губиться в повідомленнях
- Веде **таблицю Excel** або паперовий блокнот
- Не знає, скільки заробив за місяць — рахує вручну
- Платить за окремий інструмент запису, окремий для продажу товарів, ще окремий для нагадувань
- **Не продає товари** під час запису — хоча клієнт купив би

**Ринок:** ~300 000 соло-майстрів в Україні. 90% досі працюють "по старинці".

## Рішення — чим Bookit відрізняється

### Гібридний кошик (ключовий диференціатор)
Жоден конкурент — ні Booksy, ні Fresha, ні українські аналоги — не дає зробити це в одному флоу:

```
Клієнт вибирає манікюр + нарощування → обирає час →
бачить "Можливо вам сподобається: ваш улюблений гель-лак OPI" →
додає до кошика → один чек, одне підтвердження
```

Майстер отримує **оплату за роботу + продаж товару** з одного запису.

### Smart Slot Suggestion
Алгоритм аналізує, коли клієнт зазвичай записується, і підсвічує "найкращі" слоти зіркою (★). Жодного LLM — чистий scoring на основі реальних даних.

### Автоматизація під ключ
- Email-підтвердження одразу після запису
- Нагадування клієнту за 24 год (Vercel Cron + Resend)
- Telegram-сповіщення майстру в реальному часі
- Web Push для підписаних клієнтів

### Програма лояльності
Майстер налаштовує: "після 5 візитів — знижка 10%". Система сама розраховує і застосовує знижку під час запису. Клієнт бачить: "🎁 Знижка постійного клієнта -10%".

### Авто-теги клієнтів
CRM автоматично маркує клієнтів: `Новий`, `Постійний`, `Великий чек`, `Не приходив 60д` — без жодних налаштувань.

## Функціонал (реалізовано)

### Для майстра — Dashboard
| Розділ | Що вміє |
|--------|---------|
| **Огляд** | Виручка сьогодні, записи, спарклайн тижня, швидкі дії |
| **Записи** | Список з фільтрами, зміна статусу, нотатки, пагінація, email клієнту при підтвердженні/скасуванні |
| **Послуги** | CRUD з emoji, категорія, ціна, тривалість, фото, сортування ↑↓ |
| **Товари** | CRUD з фото, залишками, крос-сейл зв'язки з послугами |
| **Клієнти** | CRM: пошук, статистика, VIP-статус, авто-теги, останні записи |
| **Аналітика** | KPI місяця, бар-чарт 6 місяців, топ-5 послуг, топ-10 товарів (Pro) |
| **Лояльність** | Програми знижок: кількість візитів → % знижки |
| **Налаштування** | Фото, emoji-аватар, slug, bio, Instagram/Telegram, розклад роботи, тема |
| **Тариф** | Поточний план, перехід через WayForPay |
| **Сповіщення** | Bell з real-time підвантаженням, відкриває деталі запису |

### Для клієнта — Публічна сторінка + BookingFlow

**Публічна сторінка** (`/[slug]`) — Server-Side Rendered для SEO:
- Аватар, ім'я, рейтинг, опис, соцмережі
- Список послуг з фільтром по категоріях
- Карусель товарів
- Відгуки клієнтів
- Sticky кнопка "Записатися"

**BookingFlow** — 5-кроковий мобільний флоу:
1. **Послуги** — вибір однієї або кількох, підрахунок суми
2. **Дата/час** — слоти за розкладом майстра + зайняті. Smart Slot Suggestion: ★ на рекомендованих
3. **Товари** — крос-сейл: тільки товари, пов'язані з обраними послугами
4. **Клієнт** — автозаповнення для авторизованих, програма лояльності
5. **Підтвердження** — фінальний чек з урахуванням знижки

**Клієнтський кабінет** (`/my/bookings`):
- Майбутні та минулі записи
- Скасування з підтвердженням
- Відгук після завершеного візиту (1–5 зірок + текст)
- Повторний запис до того ж майстра

## Архітектурні рішення

### Server / Client розподіл
```
page.tsx (Server)          — fetch, auth, SEO meta, props down
*Page.tsx (Client)         — useState, TanStack Query, forms
app/api/ (Route Handlers)  — webhooks (WayForPay, Telegram), cron
app/[slug]/actions.ts      — Server Actions: notifications, email
```

### RLS скрізь
Кожна таблиця в Supabase захищена Row Level Security. `get_my_tenant_id()` прив'язує дані до конкретного майстра. Жодного витоку даних між акаунтами.

### SQL тригери
- `increment_booking_counter` — автоматично лічить `bookings_this_month` при новому записі
- `update_client_master_metrics` — оновлює статистику клієнта (`total_visits`, `total_spent`) при `completed`
- `trg_decrement_product_stock` — списує залишки товарів при `completed`

### Vercel Cron
```
0 7 * * *    →  /api/cron/reminders      (email-нагадування щодня о 9:00 Kyiv)
5 0 1 * *    →  /api/cron/reset-monthly  (скидання лічильника + downgrade)
```

## Roadmap

### Найближче (Q2 2026)
- **Реферальна система** — вірусний growth loop: клієнт запрошує майстра → обидва отримують бонус
- **Revenue Forecast** — прогноз виручки на основі трендів (linear regression, no LLM)
- **Каталог майстрів** `/explore` — повноцінний пошук з фільтрами по місту і категорії

### Середньострокове (Q3 2026)
- **Studio тариф** — мультимайстер, зведена аналітика, спільний розклад
- **Stripe/Mono** — альтернативні платіжні методи
- **Mobile App** — React Native на базі існуючого API

### Довгострокове (2027)
- **Marketplace** — клієнт шукає майстра на платформі, не тільки заходить за посиланням
- **AI-асистент** — автоматичні відповіді клієнтам, оптимізація розкладу
- **B2B (Studio+)** — великі салони, мережі

## Конкуренти

| Продукт | Де програють Bookit |
|---------|---------------------|
| **Booksy** | Немає продажу товарів. Дорогий (від $30/міс). Не адаптований під UA |
| **Fresha** | Комісія з кожного запису. Складний. Орієнтований на великі салони |
| **Setmore** | Немає товарів. Немає Telegram. Немає UA локалізації |
| **Локальні (Reservio, SimplyBook)** | Застарілий UX. Немає гібридного кошика. Немає авто-тегів |

**Наша перевага:** єдина платформа, де запис + продаж товарів + CRM + лояльність = один інструмент, адаптований під українського соло-майстра, з Telegram як першим каналом зв'язку.
