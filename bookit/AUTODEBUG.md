# BOOKIT — Ультимативний Протокол 100% Тестування та Відладки (Behavioral TDD)

**РОЛЬ:** Ти — Autonomous Lead QA Architect та Senior Full-Stack Developer.
**ЗАВДАННЯ:** Створити E2E інфраструктуру (Playwright), яка покриває КОЖНУ функцію, описану в `BOOKIT_UPTZ.md,`. Ти маєш виявити всі "тихі" баги та переписати код Next.js/Supabase до стану повної працездатності.
**МЕТА:** 100% відповідність ТЗ. Кожна задумана дія має не просто "не видавати помилку", а ФАКТИЧНО виконуватися (зберігатися в БД, оновлювати UI).


## 🛑 АРХІТЕКТУРНІ ПРАВИЛА (CRITICAL RULES)
1. **DEEP BEHAVIORAL VERIFICATION (Глибока перевірка результату):** Твої тести не мають просто клікати по кнопках. Ти ПОВИНЕН перевіряти наслідки дій. 
   - *Приклад:* Якщо клієнт записується, тест має перевірити не тільки екран "Успіх", але й програмно перевірити (через Supabase API або UI адмінки майстра), що запис *дійсно* з'явився в базі даних, а залишок товару на складі зменшився.
2. **NO GUESSING:** Працюємо за циклом: Написав тест для фічі з ТЗ -> Тест впав -> Прочитав трейс та лог бази -> Виправив код (Next.js/SQL) -> Тест пройшов.
3. **100% COVERAGE:** Ми не тестуємо "тільки критичні зони". Ти маєш послідовно написати тести для ВСЬОГО, що є в `BOOKIT_TZ.md`.

## 🤖 ФАЗА 1: Ініціалізація QA Інфраструктури
1. Прочитай `BOOKIT_TZ.md` для глибокого розуміння всіх бізнес-правил.
2. Встанови `@playwright/test`.
3. Налаштуй `playwright.config.ts` для глибокого логування (`trace: 'retain-on-failure'`, `video: 'retain-on-failure'`).
4. Налаштуй Page Object Model (POM) у папці `e2e/pages/`.
5. Створи утиліту `e2e/utils/supabase.ts` для прямих SQL-запитів до бази даних (щоб тести могли перевіряти фактичне створення записів, зміну статусів та RLS).

**-> Зупинись і відзвітуй про завершення Фази 1. Переконайся, що скрипти запуску додані.**

## 🛠 ФАЗА 2: Написання Повного Test Suite (За ТЗ)
*Створи набір тестів, який пройдеться по кожному розділу ТЗ. Тести мають бути суворими до бізнес-логіки.*

Створи наступні файли в `e2e/tests/`:
1. **`01-auth.spec.ts`:** Перевірка реєстрації майстра (всі 3 кроки, генерація slug), логіну клієнта, ізоляції ролей (майстра не пускає в клієнтську зону і навпаки), збереження сесії PWA.
2. **`02-master-core.spec.ts`:** Створення, редагування та видалення послуг/товарів. Перевірка завантаження медіа (Storage). Налаштування графіка (створення виключень `schedule_exceptions`).
3. **`03-booking-flow.spec.ts`:** Повний флоу (4 кроки). Жорстка перевірка математики "Гібридного кошика" (`total_price` = послуги + товари). Перевірка автозаповнення даних для авторизованого клієнта.
4. **`04-crm-logic.spec.ts`:** Зміна статусів запису (через UI майстра). Перевірка, що при статусі `completed` спрацьовує SQL-тригер і оновлюється таблиця `client_master_relations` (LTV, кількість візитів).
5. **`05-loyalty-reviews.spec.ts`:** Перевірка, що після 5-го візиту клієнта автоматично застосовується знижка (з `loyalty_programs`). Перевірка можливості залишити відгук та його появи на `/[slug]`.
6. **`06-referrals.spec.ts`:** Генерація інвайт-лінку клієнтом, реєстрація нового майстра за цим лінком, нарахування "Boosted Start" бонусів.
7. **`07-notifications.spec.ts`:** Перевірка запису в таблицю `notifications` при зміні статусу бронювання (емуляція тригерів).

**-> Запусти весь набір. Уважно прочитай усі помилки (де код працює "вхолосту", а де падає).**

## 🚀 ФАЗА 3: Автономна Відладка та Імплементація (The Fixing Loop)
*Тепер твоє завдання — перетворити всі червоні тести на зелені.*

**АЛГОРИТМ ДЛЯ КОЖНОГО БАГУ/ВІДСУТНЬОЇ ФІЧІ:**
1. Якщо дія не відбулася (наприклад, товар не додався до запису):
   - Перевір API Route / Server Action. Чи правильно передаються параметри?
   - Перевір RLS бази. Чи не блокується INSERT мовчки?
   - Перевір UI State (Zustand/React Query). Чи викликається інвалідація кешу після мутації?
2. Якщо фіча взагалі відсутня в коді (наприклад, знижки по лояльності):
   - Напиши необхідні SQL-міграції, Server Actions та UI-компоненти згідно з ТЗ.
3. Внеси зміни в код, перезапусти конкретний тест. Працюй, поки дія не буде виконуватися на 100% правильно.

**-> Працюй файл за файлом. Звітуй мені після успішного "позеленіння" кожного модуля.**
