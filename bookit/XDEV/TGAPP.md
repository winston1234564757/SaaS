# 📱 Telegram Mini App (TMA) Integration Log

Цей файл містить повну історію розробки, архітектурні рішення та виклики під час інтеграції Telegram Mini App у проект BookIT.

---

## 🏗️ Архітектура
- **SDK**: Використовується офіційний `https://telegram.org/js/telegram-web-app.js` + локальний фоллбек `/lib/telegram-web-app.js`.
- **Auth Flow**: 
  1. Валідація `initData` на сервері (`/api/auth/telegram`).
  2. Запит контакту через нативне вікно Telegram.
  3. Обробка контакту через Webhook (`/api/telegram/webhook`).
  4. Автоматичний вхід через Magic Link після синхронізації номера телефону.
- **SOT (Source of Truth)**: Номер телефону.

---

## 📝 Хронологія розробки

### Фаза 1: Ініціалізація та SDK
- **Проблема**: Нескінченний спіннер та помилка `SDK: NO`.
- **Причина**: Скрипт Telegram блокувався або завантажувався занадто пізно в Next.js App Router.
- **Рішення**:
    - Перехід з `next/script` на класичний `<script>` у `body` (згодом повернуто на `Script` з `beforeInteractive`).
    - Створення локального фоллбека `public/lib/telegram-web-app.js`, щоб гарантувати наявність об'єкта `window.Telegram`.
- **Виклик (Case Sensitivity)**: Виявлено, що Telegram іноді передає параметри у верхньому регістрі (`#TGWEBAPPDATA`). Додано `.toLowerCase()` валідацію.

### Фаза 2: Автентифікація та Контакти
- **Проблема**: Кнопка підтвердження номера була нечитабельною, а після натискання вилітав таймаут.
- **Причина**: Бот-вебхук ігнорував повідомлення без тексту (контакти не мали поля `text`).
- **Рішення**:
    - Повне переписування `src/app/api/telegram/webhook/route.ts` для підтримки об'єктів `contact`.
    - Реалізація автоматичного створення профілю клієнта при отриманні контакту.
    - Додано поллінг у `TelegramWelcome.tsx`, який чекає на запис у БД.

### Фаза 3: Стабілізація та Fix Web App
- **Проблема**: Звичайний браузер (Web App/PWA) застрягав на екрані завантаження Telegram.
- **Причина**: Локальний SDK завжди створював об'єкт `window.Telegram`, вводячи провайдер в оману.
- **Рішення**:
    - **Smart SDK**: Локальний скрипт тепер активується ТІЛЬКИ якщо в URL є параметри Telegram.
    - **Fast Path**: `TelegramProvider` миттєво пропускає ініціалізацію, якщо не знаходить `tgwebappdata` в URL.

---

## ⚠️ Критичні зауваження для майбутнього
1. **Z-Index & Layers**: Екран завантаження Telegram має бути поверх всього, але не блокувати звичайний Web App.
2. **Webhook Security**: Кожен `admin.update` у вебхуку має бути максимально захищеним.
3. **Identity Display**: При створенні профілю через Telegram бот використовує `first_name` + `last_name`.

---

### Фаза 4: Bot Handoff & Deep Linking
- **Проблема**: Помилка `openTelegramLink is not a function` та непрацююча кнопка повернення в боті.
- **Причина**: 
    - Локальний SDK не мав методів `openLink` та `openTelegramLink`.
    - Використання аліасу `/app` у посиланнях вимагало специфічних налаштувань у BotFather.
- **Рішення**:
    - Оновлено локальний SDK фоллбек з підтримкою методів переходу.
    - Впроваджено нативні `web_app` кнопки в Inline Keyboard бота для миттєвого та надійного запуску Mini App.
    - Додано санітизацію `botName` (видалення `@`), що усунуло помилки при відкритті посилань.

### Фаза 5: Immersive UI & Fullscreen
- **Проблема**: Додаток відкривався з "негарною" смугою зверху, попри налаштування в BotFather.
- **Причина**: Відсутність явного виклику `requestFullscreen` та невідповідність системних кольорів дизайну BookIT.
- **Рішення**:
    - Активовано `tg.requestFullscreen()` (для Telegram 8.0+).
    - Синхронізовано `headerColor` та `backgroundColor` через SDK з фірмовим Peach кольором (`#FFE8DC`).
    - Впроваджено `viewport-fit=cover` та підтримку Safe Area Insets через CSS-змінні Telegram (`--tg-content-safe-area-inset-top`).

---

## ⚠️ Критичні зауваження для майбутнього
1. **Z-Index & Layers**: Екран завантаження Telegram має бути поверх всього, але не блокувати звичайний Web App.
2. **Tailwind v4 Warnings**: В `globals.css` IDE може підсвічувати `@plugin` та `@theme` як невідомі правила. Це особливість Tailwind v4, яка не впливає на білд.
3. **Safe Areas**: Завжди використовувати `var(--tg-content-safe-area-inset-top)` для відступів під кнопки Telegram у режимі Fullscreen.
